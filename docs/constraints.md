# 长期情感陪伴能力评测约束文档

## 1. 评测目标

- 评估模型的原生长期陪伴能力
- 尽量测量模型本身，而不是产品层体验
- 重点关注多次互动中的连续性、情绪支持质量、稳定性和关系修复能力

## 2. 明确排除的因素

- 产品级持久记忆
- 产品 UI 带来的体验影响
- 产品侧额外包装的人设
- 主动提醒、主动回访等产品功能
- 检索工具或外部知识工具
- 对话中的工具调用

## 3. Pilot 固定范围

- 比较模型数：2 个
- 测试者人数：5 人
- 总 episode 数：5
- 角色扮演 episode：3
- 真实经历 episode：2
- 每个 episode 的 session 数：3
- 每个 session 建议轮数：8 到 12 轮
- session 间建议间隔：2 到 4 天，默认推荐 3 天

## 4. 实验组成

### Part A：角色扮演

- 使用半虚构角色扮演场景
- 控制方式为中等控制
- 每个 session 只规定主题、必达点、可选透露点和挑战点

### Part B：真实经历

- 使用测试者的真实情绪经历
- 仅允许低到中等强度议题
- 第一版 pilot 不纳入高风险危机场景

## 5. 历史连续性机制

本次 pilot 不依赖产品原生跨 session 记忆，而是通过统一格式的历史摘要注入来模拟连续关系语境。

## 6. 统一原则

- 所有模型使用相同的 system prompt
- history summary 注入模板一致
- episode 结构一致
- 同一 episode 下，尽量由同一测试者完成两个模型的配对比较
- 评分使用 1 到 5 分李克特量表
